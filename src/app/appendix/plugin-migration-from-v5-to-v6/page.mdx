# Plugin Migration Guide from v5 to v6

## New `ipm` CLI tool

In v6, there is a new CLI tool named `ipm` (Inkdrop Plugin Manager) for publishing plugins.

```sh
npm install -g @inkdropapp/ipm-cli

ipm configure
```

Check out [the repository](https://github.com/inkdropapp/ipm-cli) for more details.


## `@electron/remote` is deprecated

### `remote.dialog` → `inkdrop.dialog`

Before:

```js
const remote = require('@electron/remote')
const { dialog } = remote

return dialog.showOpenDialog(
  inkdrop.window,
  {
  ...
})
```

After:

```js
return inkdrop.dialog.showOpenDialog({
  ...
})
```

<Note>
The `inkdrop.window` argument is no longer required.
</Note>

### `inkdrop.window.on()`

Before:

```js
inkdrop.window.on('focus', this.handleAppFocus)
```

After:

```js
const sub = inkdrop.window.onFocus(this.handleAppFocus)

// Unsubscribe
sub.dispose()
```

## LESS is deprecated

The necessity of LESS has been less and less over the years, since CSS supports nested selectors and variables.
In v6, the app has dropped support for LESS.

- Rename `.less` to `.css`

Before:

```less
@primary-color: #ff0000;
.my-plugin {
  .component {
    color: @primary-color;
  }
}
```
After:

```css
:root {
  --primary-color: #ff0000;
}
.my-plugin {
  .component {
    color: var(--primary-color);
  }
}
```
